## ğŸ¯ Key Architectural Decisions

1. **Pub/Sub Pattern**: FilterEngine uses observer pattern for loose coupling
2. **Immutable State**: State objects are cloned on every change
3. **ES5 Syntax**: Maximum browser compatibility without transpilation
4. **Progressive Enhancement**: Core functionality works, enhanced features layer on top
5. **Mobile-First**: Responsive breakpoints from smallest to largest
6. **Accessibility-First**: ARIA, keyboard nav, and screen reader support from day one
7. **No External Dependencies**: Only Leaflet.js (already in project)
8. **LocalStorage Persistence**: Filters survive page refresh
9. **Real-Time Updates**: Map updates immediately when filters change
10. **Defensive Programming**: Try-catch blocks, null checks, fallbacks everywhere
